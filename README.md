## 为什么编写技术文档？  
* 通过编写能让人看懂的技术文档，加强自己对某技术的理解。  
* 编写技术文档，类似于将软件变成固件，知识更加稳定、可复现、可供随时调用。  
* 量变引起质变，未来当技术文档积累足够多时，一些没有意识到的知识相关性可能会被发现，使得对事物的理解更上一层楼。  
## 为什么要先编写过程日志？  
只有反思优化思维过程才能真正提升智慧，否则只是在死板地固化知识。  
**技术文档应该基于自己的思维过程**，而不能是对他人已总结知识的生搬硬套。  
## 技术文档应该在什么时候写？  
过早写：可能记录的是他人思路的拼凑，而非自己的理解体系  
过晚写：细节和顿悟时刻早已模糊，只剩下干巴巴的结论  
所以应该**动态**地写，逐步丰富完善。  
## 什么样的知识值得记录？  
从产出回报来看，当我认为记录它能在日后为我节省比现在记录所消耗的时间更多的时间时，就应当记录。
